The Genesis

This journey began with a simple question: "What are the limits of Large Language Models when it comes to creating interactive experiences?"

The story actually starts with the wertc project, where I challenged myself to build a fully functional web chat in a single standalone HTML page. That experiment taught me the power of self-contained web applications and sparked an idea.

Life Duel was born from wanting to explore LLMs in creating novel games. But why stop at one? Together with my son, we embarked on an ambitious mission: create 100 games, learn something new from each one.

Project Goals

  • Find the limits of LLMs through practical experimentation
  • Create self-contained single HTML pages (minimal exceptions allowed)
  • Learn something new from each game implementation
  • Journey to 100 games - an ambitious learning marathon
  • Father-son collaboration - making coding a shared adventure
  • Open source everything - all code available on GitHub

What This Project Is NOT

  • Using complex frameworks or servers

We keep it simple. Each game is a testament to what's possible with vanilla HTML, CSS, and JavaScript.

The Methodology

Every game is an experiment. Take Tic-Tac-Toe: while building it, I dove deep into the minimax algorithm and even trained our own ML model. Each game becomes a learning laboratory.

The magic happens in the constraints: one HTML file, no server dependencies (except for a few special cases), and always pushing the boundaries of what an LLM can help create.

The detailed stories of each game's development, challenges, breakthroughs, and lessons learned are embedded within the games themselves. Click on any game to discover the journey behind its creation.

The Two-Directional Journey

One fascinating observation while pursuing the 100-games goal: this journey moves in two directions simultaneously. We could implement very simple games and quickly move to the next one, or we could deeply improve each game by adding more levels, testing extensively, and refining the experience.

The key insight? Balance is everything. If we get stuck in the "quick and simple" direction, we end up with 100 stupid or non-playable games. If we get stuck perfecting just one game, we'll never reach 100.

This tension creates a sweet spot: each game must be complete enough to be engaging and educational, but simple enough to allow progress toward our ambitious goal. It's a constant calibration between depth and breadth.

The Adventure Continues

We're 0 games into our journey to 100. Each game teaches us something new about AI collaboration, web development, game design, or computer science concepts. Some experiments succeed brilliantly, others teach us valuable lessons about limitations.

This isn't just about building games—it's about documenting the future of human-AI collaboration in software development, one HTML file at a time.
